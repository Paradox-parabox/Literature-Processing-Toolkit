# PICOS-catcher

从研究文档自动提取PICOS框架，生成标准化的纳入/排除准则文档。

## 工作流程

```
输入文档 → 领域识别 → 逐句审计 → 提取PICOS → 识别相关领域与歧义词 → 生成准则 → 输出.md文件
```

## 第一步：领域识别

在提取PICOS之前，首先识别研究所属领域，以选择合适的PICOS框架：

### 领域判断逻辑

```
检测关键词 → 判断领域 → 选择PICOS框架
```

**数学相关领域判断**：

| 判断条件 | 领域类型 | PICOS框架 |
|---------|---------|----------|
| 含"证明""定理""收敛性""稳定性"等 | 纯数学/计算数学 | 数学PICOS |
| 含"算法""数值""离散""计算"等 | 计算数学 | 数学PICOS |
| 含"方程""模型""模拟"等 | 应用数学 | 数学PICOS |
| 含"估计""推断""统计""分布"等 | 统计学 | 数学PICOS |
| 作者单位含"数学学院""数学系" | 数学交叉学科 | 数学PICOS |
| 其他情况 | 通用领域 | 标准PICOS |

**数学PICOS框架**：见 [references/math_picos_framework.md](references/math_picos_framework.md)

**标准PICOS框架**：见 [references/picos_framework.md](references/picos_framework.md)

## 第二步：逐句审计与断言拆解

**目标**：对文本进行"显微镜式"扫描，精准识别核心方法与对比参照，避免方法稀释和关键信息遗漏。

### 执行原则（Strict Rules）

#### 1. 原子性（Atomicity）

**规则**：如果一个长句包含多个独立的学术事实，必须拆分为多个独立的PICOS要素。

**示例**：
```
原文："有限元方法矩阵装配复杂，推广到高维困难，而LBM方法边界处理简单。"

错误处理：
I: 有限元方法、LBM方法  ← 合并处理，导致方法稀释

正确处理：
C: 有限元方法（对比参照）→ 局限：矩阵装配复杂，推广高维困难
I: LBM方法（核心方法）→ 优势：边界处理简单
```

#### 2. 并列拆分（Parallel Splitting）

**规则**：句子中并列提及的方法/理论，必须为每个术语单独判断其角色（核心方法 vs 对比参照 vs 相关方法）。

**判断依据**：
- **核心方法**：正文明确采用、发展、改进的方法
- **对比参照**：正文指出其局限性、作为对标的方法
- **相关方法**：正文提及但未明确表态的方法

**示例**：
```
原文："研究多相流体输运过程的LB方法大致分为三类，即基于伪势模型的LB方法、
基于自由能模型的LB方法和基于相场理论的LB方法。"

处理：
I-核心: 相场LB方法（正文后续明确采用）
C-参照: 伪势LB模型（正文指出"含人工假设项"）
C-参照: 自由能LB模型（正文指出"仅适用于Boussinesq假设"）
```

#### 3. 深挖转折与局限（Capture Limitations）

**规则**：极度关注带有"但是"、"仅适用于"、"尚未解决"、"困难"、"挑战"等转折词的子句。这些限制条件是识别**对比参照**和**研究空白**的关键。

**转折词识别表**：

| 转折词类型 | 示例 | 含义 |
|-----------|------|------|
| **局限性** | "但是"、"然而"、"仅适用于"、"局限于" | 后面是对比参照或研究空白 |
| **困难点** | "困难"、"挑战"、"瓶颈"、"问题" | 后面是需要解决的科学问题 |
| **未解决** | "尚未"、"缺乏"、"未有"、"甚少" | 后面是研究空白 |
| **优势** | "优点"、"优势"、"能够"、"可以" | 后面是核心方法的卖点 |

**示例**：
```
原文："Fourier-Galerkin方法不适用于更一般的Dirichlet边界条件。"

处理：
C-参照: Fourier-Galerkin方法 → 局限：仅适用周期边界
P-核心: Dirichlet边界条件处理 → 研究空白/创新点
```

#### 4. 总结句升华（Summary Backing）

**规则**：段落末尾的"综上所述"、"总而言之"等总结句，需要单独提取其核心论点，作为PICOS要素的顶层概括。

**示例**：
```
原文："综上所述，尽管国内外已有不少学者围绕非局部相场模型的数值方法开展了
相关研究，但仍然有一些基础问题亟待解决..."

处理：
P-核心: 非局部相场模型的数值方法
研究空白: 基础问题亟待解决（后续列出具体问题）
```

#### 5. 禁止概括（No Summarization）

**规则**：严禁将整个段落概括为一个Intent。必须按"句"和"子句"为单位处理，宁可多采，不可漏采。

**错误示例**：
```
原文："LBM方法具有许多独特的优点：清晰的物理背景、天然的并行性、良好的扩展性。
有限元方法矩阵装配复杂。PINNs可以用于参数反演。"

错误处理：
I: LBM方法、有限元方法、PINNs  ← 概括合并，丢失关键信息

正确处理：
I-核心: LBM方法 → 优点：物理背景清晰、天然并行、良好扩展性
C-参照: 有限元方法 → 局限：矩阵装配复杂
I-核心: PINNs → 用途：参数反演
```

## 第三步：提取PICOS要素

### 数学视角PICOS（数学相关领域）

| 要素 | 名称 | 定义 | 必要性 |
|------|------|------|--------|
| **P** | Problem | 问题域（数学问题+可选应用背景） | 必需 |
| **I** | Instrument | 方法/工具（具体数学方法或算法） | 必需 |
| **C** | Comparison | 对比参照（与现有方法/结果对比） | 可选 |
| **O** | Outcomes | 产出（数学产出+可选应用产出） | 必需 |
| **S** | Study design | 研究范式（理论/算法/实验等） | 必需 |

### 核心方法 vs 对比参照判断流程

```
识别到方法M
    ↓
检查正文对M的态度
    ├── 明确采用/发展/改进 → I-核心方法
    ├── 指出局限性/作为对标 → C-对比参照
    ├── 提及但未明确表态 → I-相关方法（低优先级）
    └── 作为数据来源 → S-特殊类型（如实验数据）
```

详细说明见 [references/math_picos_framework.md](references/math_picos_framework.md)

## 第四步：识别相关领域与歧义词

### 4.1 识别相关领域

根据研究对象和方法，推断可能相关的学科领域。

### 4.2 识别歧义词（同词异义）

在相关领域内，识别可能产生歧义的关键词。

---

## 第五步：生成纳入/排除准则

### 纳入标准

基于PICOS要素，生成明确的纳入标准。

### 排除标准

**重要**：排除标准应基于**研究特征**而非领域标签！

### 对比参照警示

**必须**在准则中明确标注哪些方法是对比参照，避免检索时被海量传统方法文献淹没。

---

## 第六步：输出标准化文档

生成 `PICOS_criteria_{timestamp}.md` 文件，包含：

1. **文档元信息** - 来源、提取时间、领域类型
2. **PICOS框架** - 完整的五要素分析
3. **核心方法 vs 对比参照** - 明确区分，避免方法稀释
4. **相关领域** - 推断的相关学科领域
5. **歧义词警示** - 同词异义分析表
6. **纳入标准** - 明确的检查清单
7. **排除标准** - 基于研究特征的排除条件
8. **筛选建议** - BM25阈值、引用数阈值等
