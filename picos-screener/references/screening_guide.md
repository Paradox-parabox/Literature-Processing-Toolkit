# 筛选判断指南

## 一、通用筛选原则

### 1.1 核心原则

```
1. 严格遵循PICOS标准：所有判断必须基于PICOS标准文件
2. 宁可纳入，不可遗漏：不确定的论文进入下一阶段
3. 记录原因：每篇排除论文必须有明确原因
4. 保持一致：同一标准贯穿始终
```

### 1.2 PICOS要素动态提取

**关键**：筛选指南不预设具体关键词，而是从PICOS标准文件中动态提取：

| 要素 | 提取内容 | 用途 |
|------|---------|------|
| **P** | 问题域描述、应用背景 | 判断研究问题相关性 |
| **I** | 核心方法、关键词 | 判断方法相关性 |
| **C** | 对比参照方法 | 识别排除目标 |
| **O** | 期望产出 | 辅助判断 |
| **S** | 研究范式 | 判断研究类型 |
| **纳入标准** | 必须满足的条件 | 纳入检查清单 |
| **排除标准** | 排除条件 | 排除检查清单 |
| **歧义词** | 同词异义警示 | 避免误判 |

## 二、去重处理

### 2.1 去重必要性

论文列表可能包含重复记录，原因包括：
- 同一论文被多个数据库收录
- 不同搜索策略返回相同论文
- 数据导出过程中的重复

### 2.2 去重策略

```
去重步骤：
1. 提取标题字段，转换为小写
2. 去除首尾空白字符
3. 按标题去重，保留第一条记录
4. 记录去重前后的数量差异

输出：
- 原始论文数：N篇
- 去重后论文数：M篇
- 去除重复：N-M篇
```

## 三、两阶段筛选流程

### 3.1 第一阶段：标题+摘要筛选

**目标**：快速识别明显相关或明显不相关的论文

**输入**：
- 论文列表（CSV格式，含title字段）
- PICOS标准文件

**处理流程**：

```
对于每篇论文：

Step 1: 提取PICOS关键词
  - 从I要素提取核心方法关键词
  - 从C要素提取对比参照关键词
  - 从P要素提取问题域关键词

Step 2: 标题检查
  - 标题包含I要素关键词？ → 可能纳入
  - 标题包含C要素关键词？ → 可能排除
  - 标题无关键词？ → 继续检查摘要

Step 3: 摘要检查
  - 摘要第一句：研究目的是什么？
  - 摘要中间：采用什么方法？
  - 摘要最后：得出什么结论？

Step 4: 记录匹配信息
  - 记录匹配的关键词列表
  - 记录匹配类型（关键词匹配/SEED关联）
  - 提取摘要中包含关键词的句子

Step 5: 给出判断
  ├── 纳入 → 进入第二阶段
  ├── 排除 → 记录排除原因
  └── 不确定 → 进入第二阶段（待定）
```

**快速判断技巧**：

```
优先检查：
1. 标题中的核心关键词
2. 摘要第一句（研究目的）
3. 摘要最后一句（研究贡献）

跳过：
- 详细的方法描述
- 具体的数值结果
- 详细的讨论部分
```

### 3.2 第二阶段：全文筛选

**目标**：详细验证不确定或可能纳入的论文

**适用对象**：
- 第一阶段"纳入"的论文
- 第一阶段"待定"的论文

**处理流程**：

```
对于每篇论文：

Step 1: 获取更多信息
  - 检查本地是否有PDF文件
  - 通过DOI/URL获取扩展信息
  - 使用WebSearch搜索论文标题

Step 2: 验证研究问题
  - 是否匹配P要素定义的问题域？
  - 是否涉及PICOS指定的应用背景？

Step 3: 验证研究方法
  - 是否采用I要素定义的核心方法？
  - 方法创新点是什么？

Step 4: 检查排除条件
  - 是否属于C要素定义的对比参照？
  - 是否满足排除标准？

Step 5: 最终判断
  ├── 纳入 → 进行质量评分
  └── 排除 → 记录详细排除原因
```

## 四、匹配类型识别

### 4.1 关键词匹配 vs SEED关联

**区分方法**：

```
检查CSV中的Directive_Type字段：
├── "SEED" → SEED关联
│   └── 显示关联的种子论文（Seed_Paper字段）
└── "QUERY" 或其他 → 关键词匹配
    └── 显示匹配的关键词列表
```

### 4.2 输出格式

| 匹配类型 | 匹配依据 | 说明 |
|---------|---------|------|
| 关键词匹配 | keyword1, keyword2, ... | 论文标题/摘要包含PICOS关键词 |
| SEED关联 | 种子论文: {Seed_Paper} | 论文通过滚雪球方式获得 |

## 五、依据映射与提取

### 5.1 正文依据映射

**目的**：显示论文与立项依据的相关性

**方法**：

```
对于每篇纳入论文：
1. 获取其匹配的关键词列表
2. 从关键词-正文映射中查找对应的正文依据
3. 输出正文原文

映射构建：
从PICOS标准文件中提取关键词与正文原文的对应关系
```

### 5.2 摘要依据提取

**目的**：快速了解论文具体贡献

**方法**：

```
对于每篇纳入论文：
1. 获取其摘要文本
2. 按句子分割（句号、问号、感叹号）
3. 遍历每个句子，检查是否包含匹配关键词
4. 提取包含关键词的句子（最多2-3句）
5. 如句子过长，截断至200字符
```

## 六、排除原因分类

### 通用排除原因代码

| 代码 | 原因 | 判断依据 |
|------|------|---------|
| **E1** | 研究问题不相关 | 不匹配P要素 |
| **E2** | 研究方法不相关 | 不匹配I要素 |
| **E3** | 属于对比参照方法 | 匹配C要素 |
| **E4** | 研究类型不符 | 不匹配S要素 |
| **E5** | 其他原因 | 需具体说明 |

### E1 - 研究问题不相关

**判断标准**：
- 研究问题与PICOS的P要素无关
- 不涉及PICOS定义的问题域
- 不涉及PICOS指定的应用背景

**记录格式**：
```
排除原因：E1 - 研究问题不相关
具体说明：[说明为什么不匹配P要素]
```

### E2 - 研究方法不相关

**判断标准**：
- 未采用PICOS的I要素定义的核心方法
- 方法与I要素完全不相关

**记录格式**：
```
排除原因：E2 - 研究方法不相关
具体说明：[说明为什么不匹配I要素]
```

### E3 - 属于对比参照方法

**判断标准**：
- 研究的是PICOS的C要素定义的对比参照方法
- 正文明确指出该方法的局限性

**记录格式**：
```
排除原因：E3 - 属于对比参照方法
具体说明：[说明匹配哪个C要素]
```

### E4 - 研究类型不符

**判断标准**：
- 论文类型与PICOS的S要素不符
- 如：纯综述、纯实验、纯理论（视S要素定义）

**记录格式**：
```
排除原因：E4 - 研究类型不符
具体说明：[说明为什么不匹配S要素]
```

### E5 - 其他原因

**常见情况**：
- 全文无法获取
- 语言不符
- 数据不完整
- 重复文献

**记录格式**：
```
排除原因：E5 - 其他原因
具体说明：[具体说明原因]
```

## 七、质量评分标准

### 7.1 方法创新性（1-5分）

| 分数 | 标准 |
|------|------|
| 5 | 提出全新方法/算法 |
| 4 | 显著改进现有方法 |
| 3 | 适度改进现有方法 |
| 2 | 应用现有方法解决新问题 |
| 1 | 简单应用现有方法 |

### 7.2 相关性程度（1-5分）

| 分数 | 标准 |
|------|------|
| 5 | 完全匹配PICOS所有要素 |
| 4 | 匹配PICOS大部分要素 |
| 3 | 匹配PICOS核心要素 |
| 2 | 部分匹配PICOS要素 |
| 1 | 边缘相关 |

### 7.3 引用数权重（1-5分）

| 分数 | 标准 |
|------|------|
| 5 | 引用数 ≥ 100 |
| 4 | 引用数 50-99 |
| 3 | 引用数 20-49 |
| 2 | 引用数 10-19 |
| 1 | 引用数 < 10 或无数据 |

### 7.4 综合评分计算

```
综合评分 = (方法创新性 + 相关性程度 + 引用数权重) / 3

优先级划分：
- 高优先级：综合评分 ≥ 4
- 中优先级：综合评分 3-3.9
- 低优先级：综合评分 < 3
```

## 八、歧义词处理

### 8.1 识别歧义词

从PICOS标准文件的"歧义词警示"部分提取：

| 词汇 | 目标领域含义 | 其他领域含义 | 处理方法 |
|------|-------------|-------------|---------|
| {word} | {target_meaning} | {other_meaning} | 检查上下文 |

### 8.2 处理策略

```
遇到歧义词时：
1. 检查标题中的其他关键词
2. 检查摘要中的领域术语
3. 检查作者单位/期刊
4. 如仍不确定，标记为"待定"
```

## 九、输出规范

### 9.1 筛选报告格式

**两表格格式**：

```
表一：纳入论文清单（按优先级排序，仅显示中高优先级）
- 高优先级论文（综合评分 ≥ 4）
- 中优先级论文（综合评分 3-3.9）
- 低优先级论文不计入最终表格，但仍计入PRISMA统计

每篇论文包含：
| 序号 | 论文标题 | 匹配类型 | 匹配依据 | 正文依据 | 摘要依据 |

表二：排除论文统计
| 排除原因 | 数量 | 说明 |
```

### 9.2 CSV输出字段

```
必需字段：
- id: 论文序号
- title: 标题
- decision: 纳入/排除
- match_type: 匹配类型（关键词匹配/SEED关联）
- match_detail: 匹配依据
- body_evidence: 正文依据
- abstract_evidence: 摘要依据
- quality_score: 质量评分（纳入论文）
- exclude_reason: 排除原因代码
- exclude_detail: 排除原因详细说明

可选字段：
- authors, year, journal, doi, url等
```

## 十、批量筛选效率优化

### 10.1 分批处理

```
每批处理：20篇论文
批次间隔：报告进度
总进度显示：X/N篇已处理
```

### 10.2 进度报告

```
筛选进度报告：
- 已处理：XX篇
- 纳入：XX篇
- 排除：XX篇
- 待定：XX篇
- 进度：XX%
```

### 10.3 不确定论文处理

```
不确定时：
1. 标记为"待定"
2. 记录不确定原因
3. 进入第二阶段详细分析
4. 不在第一阶段纠结
```
